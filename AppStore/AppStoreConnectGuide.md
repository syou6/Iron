# App Store Connect 設定ガイド

## 1. 事前準備

### 必要なもの
- [ ] Apple Developer Program メンバーシップ（年間$99）
- [ ] App Store Connect アカウント
- [ ] Xcodeで署名済みのアプリ

---

## 2. App Store Connect でアプリを作成

### 2.1 新規アプリの作成

1. [App Store Connect](https://appstoreconnect.apple.com) にログイン
2. 「マイApp」→「+」→「新規App」
3. 以下を入力：

| 項目 | 値 |
|------|-----|
| プラットフォーム | iOS |
| 名前 | Nan Ton? - 筋トレ記録 |
| プライマリ言語 | 日本語 |
| バンドルID | com.shodev.nanton |
| SKU | nanton-v1 |

---

## 3. アプリ情報の入力

### 3.1 基本情報

**App Information タブ:**

| 項目 | 値 |
|------|-----|
| 名前 | Nan Ton? - 筋トレ記録 |
| サブタイトル | シンプルで強力な筋トレ記録アプリ |
| カテゴリ | ヘルスケア/フィットネス |
| セカンダリカテゴリ | ライフスタイル |
| コンテンツ権利 | 自分が権利を所有 |
| 年齢制限 | 4+ |

### 3.2 価格と配信地域

**Pricing and Availability タブ:**

- 価格: 無料
- 配信地域: 日本（または全世界）

---

## 4. バージョン情報

### 4.1 スクリーンショット

**必要なサイズ:**

| デバイス | サイズ | 必要枚数 |
|----------|--------|----------|
| iPhone 6.7" | 1290 x 2796 | 1-10枚 |
| iPhone 6.5" | 1284 x 2778 | 1-10枚 |
| iPhone 5.5" | 1242 x 2208 | 1-10枚 |
| iPad Pro 12.9" | 2048 x 2732 | オプション |

**撮影するおすすめ画面:**
1. ホーム画面（3Dマッスルヒートマップ）
2. ワークアウト記録画面
3. 統計/グラフ画面
4. エピック・マイルストーン
5. 種目一覧

### 4.2 プロモーションテキスト（170文字）

```
Nan Ton?（何トン?）は、あなたの筋トレを完全サポート。ワークアウトの記録、進捗の可視化、3Dマッスルヒートマップで鍛えた部位を確認。累計挙上重量で「自由の女神を持ち上げた！」などの達成感も味わえます。
```

### 4.3 説明文

`AppStoreDescription_ja.md` の内容をコピー

### 4.4 キーワード（100文字）

```
筋トレ,トレーニング,ワークアウト,記録,ジム,フィットネス,筋肉,ベンチプレス,スクワット,ダンベル,バーベル,1RM,進捗,ログ
```

### 4.5 サポートURL

GitHubリポジトリまたはWebサイトのURL

### 4.6 プライバシーポリシーURL

`PrivacyPolicy_ja.md` をWebサイトにホストしてURLを入力

## 5. App Privacy（プライバシー詳細）

App Store Connect の「App Privacy」セクション：

### 収集するデータ

**推奨設定（本アプリの場合）:**

| カテゴリ | 収集 | 理由 |
|----------|------|------|
| 連絡先情報 | いいえ | - |
| 健康とフィットネス | はい（HealthKit使用時） | アプリ機能 |
| 位置情報 | いいえ | - |
| 識別子 | いいえ | - |
| 使用状況データ | いいえ | - |
| 診断 | いいえ | - |

**「データは収集していません」を選択可能な場合:**
- アナリティクスなし
- 広告なし
- ユーザートラッキングなし

---

## 6. ビルドのアップロード

### 6.1 Xcodeからアップロード

1. Xcode でプロジェクトを開く
2. Product → Archive
3. Distribute App → App Store Connect
4. Upload

### 6.2 TestFlight でテスト

1. App Store Connect でビルドを確認
2. 「TestFlight」タブでテスターを追加
3. 内部テスト → 外部テスト

---

## 7. 審査申請

### 7.1 チェックリスト

- [ ] すべてのメタデータ入力完了
- [ ] スクリーンショットアップロード完了
- [ ] プライバシーポリシーURL設定
- [ ] ビルドアップロード完了
- [ ] TestFlightテスト完了

### 7.2 審査提出

1. 「審査用に提出」をクリック
2. 広告ID（IDFA）: 使用していない
3. 輸出コンプライアンス: 暗号化は標準のHTTPSのみ
4. コンテンツ権利: 確認

### 7.3 審査時間

- 通常: 24-48時間
- 最大: 数日〜1週間

---

## 8. よくある却下理由と対策

| 却下理由 | 対策 |
|----------|------|
| クラッシュ | TestFlightで十分テスト |
| 機能不足 | 最低限の機能が動作することを確認 |
| メタデータ不正確 | スクショと実際の画面が一致 |
| プライバシー | HealthKit使用理由を明記 |

---

## 9. 承認後

1. リリース日を設定（即時 or 予約）
2. App Storeで公開を確認
3. App Analytics で指標を確認
